핵심 변경: 렌더링 전략

문제: Ink Box border

  • flexGrow가 터미널 전체를 못 채움
  • border 문자(┌┐└┘)가 너비 계산에 포함되어 불균형
  • 결과: 왼쪽 쏠림, 불균형 패널

해결: 직접 컬럼 계산

  • useStdout().columns로 터미널 너비 가져오기
  • 문자열을 .padEnd(columnWidth)로 강제 채우기
  • 수평선 '─'.repeat(columns)로 전체 너비 보장
  • Box border 삭제 → 섹션 헤더 + hrule로 구분

구현 전략

1. useStdout().columns — 터미널 너비 가져오기
2. HRule 컴포넌트 — '─'.repeat(columns) 으로 전체 너비 수평선
3. Row 컴포넌트 — 좌/우 컬럼을 고정 비율로 분할 (예: Math.floor(cols * 0.5))
4. Box border 전면 삭제 — borderStyle="round/single" 안 씀
5. 섹션 헤더 — SECTION NAME 라벨만으로 구분
6. padEnd 강제 — 모든 텍스트 행을 컬럼 너비에 맞춰 패딩
